# Scenario Report Cards

This directory holds short, human-readable summaries for each recovery scenario.

Each report card is intentionally lighter than the scenario spec. It should answer:

- what problem the scenario is trying to prove or protect against
- where the scenario is most likely to fail
- how the harness makes that failure easier to understand and recover from

Each report card should link back to:

- the main scenario spec
- the latest clean run report, if one exists
- any shared technical references that matter for understanding the scenario boundary
